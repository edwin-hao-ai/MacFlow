# MacFlow 产品讲解视频脚本

> 总时长 ~5 分钟｜适合 YouTube / B 站 / X 短视频剪辑
> **加粗** = 加重语气，*斜体* = 停顿或放缓
> `[镜头]` = 画面/演示提示

---

## 开场痛点（约 40 秒）

`[镜头：Mac 桌面，活动监视器打开，密密麻麻的进程列表]`

> 你的 Mac 是不是越用越卡？

`[切：风扇狂转的特写，或 Docker 图标 + 一堆容器]`

> 关掉一堆 App，风扇还是嗡嗡响。打开活动监视器，一长串进程看也看不懂——**到底哪个能杀，哪个不能动？**

`[切：访达里 ~/Library/Caches 文件夹，巨大的容量数字]`

> 磁盘空间一天比一天少。你知道 npm 缓存有 5 个 G，Docker 镜像囤了 30 个 G，Xcode DerivedData 早就该清——*但每次都要打开终端，敲一长串命令，记不住、不敢清、清错了项目重新拉一晚上。*

`[切：CleanMyMac 启动画面，订阅价格弹窗]`

> 用 CleanMyMac？**两百多兆的 Electron 应用**，每年还要付三十多刀订阅，扫一次系统它自己先占满 500MB 内存。更别提它**根本不懂开发者**——npm、Docker、Xcode 这些它一个都管不了。

`[黑屏 1 秒]`

> *所以我做了 MacFlow。*

## 功能 1：智能扫描（约 40 秒）

`[镜头：点击「智能扫描」，环形进度条转起来，几秒后跳出结果]`

> 打开 MacFlow，第一屏就是智能扫描。**一键，三秒。**

`[镜头特写：CPU / 内存 / 磁盘三个环形图，下面列出可清理项]`

> 它会同时扫描你的**进程、缓存、应用残留**，然后告诉你——

> 这台 Mac 现在 **能清出多少空间，能释放多少内存**。

`[镜头：鼠标 hover 在每一项上，显示详情]`

> 每一项都标了来源、大小、风险等级。**默认勾选的，全部是重新获取成本不到 5 分钟的安全项。**真正的大件，比如完整的 Docker 镜像、几十 G 的 node_modules，**默认不勾**——你自己决定。

> 然后，**一个按钮，全部搞定。**

---

## 功能 2：进程管理（约 45 秒）

`[镜头：切到「进程管理」标签，进程列表带应用图标显示]`

> 进程管理这块，我做得比活动监视器**狠一点**。

`[镜头：滚动列表，几个 node、Electron Helper 高亮]`

> 它会自动揪出三类问题进程——**僵尸进程**、**长时间空闲的内存大户**、**你早就忘记还在跑的后台 App**。

`[镜头特写：node 进程占 1.2GB，标红，旁边有「3 小时未活跃」标签]`

> 比如这个 node 进程，**占了 1.2 个 G，三小时没动过**。这种就是典型的——你切走去做别的事，它就一直挂在那儿吃内存。

`[镜头：点击「终止」按钮，绿色对勾动画]`

> 杀进程的时候，**先发 SIGTERM 软终止，等三秒不响应才 SIGKILL**——这是 Unix 该有的礼貌。

> 而且，**正在前台用的 App 一律不扫描**，系统核心进程**默认隐藏**——你不会一不小心把 Finder 杀掉。

## 功能 3：应用程序 + 应用卸载（约 50 秒）

`[镜头：切到「应用卸载」标签，按图标排列的应用宫格]`

> 这是 v0.1.1 新加的——**应用卸载**。

`[镜头：搜索框输入 "Xcode"，选中后右侧出现残留文件列表]`

> 你直接拖到废纸篓的那种「卸载」是**假卸载**。Xcode 删了，**~/Library 里几个 G 的 DerivedData 还在**。VS Code 删了，扩展、配置、缓存全留下。

> MacFlow 知道这事。

`[镜头特写：残留列表展开，~/Library/Developer/Xcode/DerivedData 显示 8.4 GB]`

> 我内置了 **Xcode、VS Code、JetBrains 全家桶、Docker Desktop、Android Studio** 的深度清理规则。卸载的时候，**它会把每个应用的隐藏巢穴一个个挖出来**。

`[镜头：点击卸载，进度条，完成]`

> *而且——**所有删除都进废纸篓，不是 rm -rf。***

> 后悔了？十秒内能捞回来。这是 CleanMyMac **绝对不敢做**的事，因为他们怕你看到自己删了什么。

---

## 功能 4：缓存清理（开发者专属，约 40 秒）

`[镜头：切到「缓存清理」，列出 npm / pnpm / yarn / Docker / Homebrew / Xcode / Cargo / pip / Go]`

> 这一块，**普通用户可以跳过——这是给开发者的**。

`[镜头特写：每个工具旁边的容量数字，npm 2.7GB、pnpm 718MB、Docker 镜像 14GB]`

> 九个开发工具，**每个都用各自的官方命令清理**——不是无脑 rm。npm 用 `npm cache clean`，Docker 用 `docker prune`，Homebrew 用 `brew cleanup`。

> *为什么这事重要？*

> 因为**乱删开发缓存是会出事的**——你正在 build 的时候缓存被清掉，整个工程崩。

`[镜头：MacFlow 检测到 docker 正在运行，对应项标灰，有提示]`

> 所以 MacFlow 在动手之前**先检查这些工具是不是在跑**。`npm install` 进行中？跳过 npm 缓存。Xcode 正在编译？跳过 DerivedData。

> **这个细节，CleanMyMac 没有，brew 自己也没有。**

## 功能 5：操作历史（约 25 秒）

`[镜头：切到「历史记录」标签，列表向下滚动展示]`

> 每一次操作，**全部本地记录**。

`[镜头特写：一条记录显示 "OpenClaw · 应用卸载 · +85.3 MB · 1 天前"]`

> 时间、对象、释放空间、成功失败——**全部可追溯**。

> 不上传，不联网，**只在你的 Mac 上**。

> 想知道前天到底删了什么？拉一下就看到。这是清理工具该有的**透明度**。

## 安全 & 隐私（约 30 秒）

`[镜头：切到产品对比表，三列：MacFlow / CleanMyMac / 活动监视器]`

> 我把这个产品做出来，最在意的就是**安全**。

`[镜头特写：逐条高亮]`

> - **零账号**——不要登录，不要邮箱
> - **零联网**——除了检查更新，**完全不发任何网络请求**。你可以打开 Little Snitch 验证
> - **零云端**——所有数据躺在你 Mac 的 SQLite 里
> - **路径白名单**——MacFlow 只能从**写死的几个安全目录**删东西。你的家目录、文稿、桌面、下载——**永远碰不到**

> *这不是营销话术，是代码层面的硬约束。*
